# ui-02 — Grid virtualization (windowed rendering)

**Commit:** `4c1126b` — "Virtualize the J-Space grid: render only the visible
row window".
**Code:** `web/index.html` (`renderGrid`, `buildGridRowTr`,
`onGridBodyScroll`, `_gridSpacerRow`).

## The problem

Reported symptom: *"when the content length is too long, the first-loaded
content in the J-Space area disappears and never reloads."*

Forensics (headless) showed the row data and DOM were **fully intact** even
after scrolling — zero stale rows. So it was not a data or DOM bug. The real
cause: at chat lengths the grid becomes a single very tall table (measured
**4,115 rows × ~36 px ≈ 148,000 px** tall), which **exceeds browser
rasterization limits**. Regions scrolled out and back into view are simply
never repainted — the pixels, not the cells, go missing. This is why the
first-loaded content looked gone forever.

(ui-01's append-only rendering made per-token cost constant but still kept
*every* row in the DOM, so it could not help here.)

## The fix

Virtualize: keep only the visible row window in the DOM.

- `renderGrid` renders `~2 × GRID_ROW_BUFFER` (≈60) rows around the viewport,
  between two spacer rows whose heights preserve exact scroll geometry.
- A rAF-throttled scroll handler (`onGridBodyScroll`) re-renders the window
  when the viewport nears a window edge.
- Row height is calibrated from a real rendered row.
- Far cell-selection jumps instantly to the target's window, then
  smooth-centers (so a selection thousands of rows away doesn't
  smooth-scroll across the whole table).

This supersedes ui-01's append fast-path with something strictly better:
constant per-token render cost **and** constant paint area at any length.

## Result (headless, synthetic 5,000-row grid)

- **65 DOM rows** rendered instead of ~325,000 cells; correct 180,000 px
  scroll geometry.
- Top / middle / bottom scroll cycles all render content (the reported bug is
  gone).
- Selecting row 2,500 is found and centered.
- A live stream follows the bottom with populated rows and coherent text.

## Related front-end fixes from the same period

Not grid-perf but shipped alongside while hardening the viewer:

- **Session data loss** (`9fa40d6`): the quota trim dropped *prefill*
  snapshots (which carry whole messages' grid rows); trim now preserves all
  prefills and a full-fidelity server-side autosave keeps the complete state.
- **Mojibake** (`0f21a6f`): byte-level BPE split CJK/emoji across tokens;
  per-token `decode([tid])` turned each fragment into U+FFFD. Streaming now
  emits UTF-8-safe cumulative segments.
- **Temperature sampling** (`b56b02c`): `mx.random.categorical` was fed
  softmax probabilities instead of logits — near-uniform sampling over the
  248k vocab (multilingual gibberish) at any `temp > 0`. Not a perf fix, but
  found during this hardening pass.
